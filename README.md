## Collaboration Flow

1. Open an Issue for each requirement, idea, bug, or task.
2. Discuss scope and acceptance criteria in the Issue before coding.
3. Create a feature branch from `main`.
4. Submit a Pull Request when the change is ready for review.
5. Merge only after review and basic verification.

## Suggested First Issues

- Define target users and usage scenarios.
- Decide the MVP scope.
- Choose the technical stack.
- Define project milestones and team responsibilities.
