# CalliLens 答辩问答

## 1. 这个项目解决什么问题？

它把传统书法作品做成可交互的数字导览。观众不仅能看高清原图，还能通过 3D 平滑浮雕观察墨色轻重、飞白和线条厚薄，并通过 RAG 问答理解作者、时代、作品背景和技术实现。

## 2. 为什么选赵孟頫《光福重建塔记》？

这件作品是赵孟頫晚年行书长卷，书于至治元年，即 1321 年，现藏上海博物馆。它既有明确的馆藏事实，也有长卷形式和历史背景，适合做横向浏览、3D 观察和知识问答。

## 3. 3D 是怎么实现的？

流程是：

```text
原图 → OpenCV 灰度化/去噪/二值化 → 墨迹 mask → height map → Three.js 3D 渲染
```

Three.js 中纸面作为底座，长卷平面根据 height map 做 Z 轴位移，形成平滑浮雕。墨色越重，高度越明显；飞白和细线更低、更薄。

## 4. 这是不是恢复真实笔顺？

不是。系统只根据图像里可见的墨迹深浅生成空间高度，不恢复真实书写顺序，也不判断作者真实用力。

## 5. 为什么现在使用平滑浮雕？

老师反馈过于抽象的悬浮效果不适合当前展示，所以当前版本回到平滑浮雕。浮雕更容易解释“墨迹深浅转高度”：重墨区域更突出，起笔、飞白和细线更低，整体仍贴近纸面。

## 6. OpenCV 做了什么？

OpenCV 负责图像处理：读取书法图像、灰度化、去噪、二值化、提取墨迹 mask，并生成 height map。它提供可见墨迹数据，不负责审美判断。

## 7. Three.js 做了什么？

Three.js 负责浏览器 3D 场景：创建纸面底座、浮雕长卷平面、相机、灯光、缩放、水平移动和拖拽旋转。它把 OpenCV 输出的图像数据转成可观察的空间效果。

## 8. RAG 是什么？

RAG 是“检索增强生成”。用户提问后，系统先从本地知识库检索相关资料，再根据资料组织回答，并显示来源。

## 9. 为什么不用 AI 直接回答？

直接问 AI 容易出现没有依据的发挥。RAG 能把回答限制在作品资料、术语解释、技术说明和老师任务要求范围内，更适合课程答辩和资料溯源。

## 10. 没有 API key 能不能运行？

可以。默认使用本地 RAG 检索和模板化回答，不需要付费 API。API 只是可选增强，用于把本地检索结果润色得更自然。

## 11. 现在有数据库吗？

当前没有 MySQL、SQLite 或向量数据库。RAG 使用本地文件型知识库：作品列表在 `data/works.json`，默认作品知识片段在 `data/work_003/knowledge.json`，通用知识库在 `knowledge/*.md`。后端每次回答问题时读取这些文件并做关键词检索。

## 12. 管理员上传有什么用？

管理员可以上传新书法作品，填写作品信息、背景和来源。系统会生成原图、缩略图、mask、height、3D 数据和知识库片段，让新作品进入作品库和 RAG 检索范围。

## 13. 为什么不开放普通用户上传？

第一版是课堂演示系统，普通用户上传图片质量不可控，可能导致处理失败或展示效果不稳定。先做管理员上传更稳。

## 14. 老师新文档 InkVerse / WebXR 怎么融合？

当前版本是稳定 Desktop Demo，负责作品、图像处理、平滑浮雕 3D 和 RAG。本分支先做 InkVerse Lite，把 Original、Enter the Ink、Follow the Qi、Return 串成短体验链；完整 WebXR、沿笔画穿行和进入留白空间放到后续阶段。RAG 负责解释每个体验节点的文化背景和技术依据。

## 15. 项目边界是什么？

系统不自动评价书法水平，不判断真伪，不恢复真实笔顺，不声称自动识别气韵。它只提供可见图像线索、3D 展示和有来源的导览解释。
