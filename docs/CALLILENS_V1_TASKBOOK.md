# CalliLens V1/V1.5 项目任务书

## 1. 项目定位

CalliLens 是一个面向普通观众的中国书法鉴赏导览 Demo。它的目标不是判断一幅字好不好，而是帮助新手把“气脉”“虚实”“笔墨节奏”等抽象概念，连接到画面里可以看见的证据。

当前版本固定使用 `data/work_003` 作为主样例。

一句话定位：

> OpenCV 提供可见证据，人工标注确认审美解释，前端把二者组织成可探索、可反思的观看流程。

## 2. 设计目标

本项目根据老师 PDF 中的 DG1-DG5 组织 V1。

### DG1: Anchor abstract concepts in visible evidence

系统必须把抽象概念锚定到具体视觉证据上，例如笔画方向、粗细变化、墨色浓淡、留白区域、字距和行距。页面不能只说“这里有气脉”，而要说明它为什么像一个连续的观看趋势。

### DG2: Support layered viewing

用户应该可以在原作、OpenCV 辅助图层、人工导览标注和局部证据之间切换。默认保持原作干净，不让系统解释一开始就压住观看。

### DG3: Preserve aesthetic openness

所有标注和高亮都应可开关。系统只给观察线索和候选证据，不输出唯一答案，不做审美评分。

### DG4: Enable reflective articulation

系统不只展示解释，还要让用户写下自己的观察。V1 页面需要提供“我的观察”输入区，鼓励用户把看到的形式特征转化为自己的语言。

### DG5: Use spatial visualization only when it clarifies invisible processes

空间可视化只在有助于说明静态图像里难以感知的运动、节奏和生成过程时使用。V1 不做 AR/VR/复杂 3D，而使用可切换图层、局部高亮和点击探针说明观看路径与空间关系。

## 3. V1 功能范围

V1 必须完成：

- 展示 `work_003` 原作局部。
- 支持 OpenCV 图层切换：原作、骨架、黑白墨迹、留白候选、粗细热力、墨色热力。
- 展示人工确认的代表性观察点。
- 点击观察卡片时，只高亮一个对应位置。
- 每个观察点包含三段解释：形式描述、感知经验、美学解释。
- 支持浏览器语音导览。
- 支持用户写下自己的观察。

V1 不做：

- 不训练模型。
- 不自动判断书法好坏。
- 不自动判断真实气韵。
- 不还原真实笔顺。
- 不检测真实书写力道。
- 不做大规模作品库。

## 4. V1.5 局部证据探针

V1.5 增加一个轻量半自动功能：

> 用户点击图像任意位置，系统分析附近局部窗口，生成算法辅助观察提示。

探针可以计算：

- 墨迹比例：附近墨迹是否密集。
- 留白比例：附近空白是否明显。
- 留白候选：是否靠近 OpenCV 标出的空白候选区域。
- 粗细变化：笔画粗细热力是否变化明显。
- 墨色变化：墨色或明暗是否变化明显。

探针只能输出候选提示，例如：

- “这里可能适合作为虚实/疏密平衡观察候选。”
- “辅助图提示附近有较明显空白通道。”
- “这里的粗细或墨色变化可作为笔墨轻重线索。”

探针不能输出：

- “系统识别出气韵。”
- “这里一定是气脉。”
- “这幅字写得好/不好。”
- “检测到真实力道。”

## 5. 数据边界

V1 主数据接口仍然是：

```text
data/work_003/annotation.json
```

OpenCV 脚本输出图层：

```text
original.png
binary.png
skeleton.png
stroke_width.png
ink_density.png
void_candidates.png
```

人工标注负责：

- 气脉路径。
- 虚实区域。
- 笔墨区域。
- 三段解释文本。

前端负责：

- 读取 JSON 和图像。
- 绘制导览高亮。
- 计算局部探针指标。
- 展示观察说明和用户反思。

## 6. 小组分工建议

- 资料与作品：确认图片来源，维护 `work-info.md`。
- OpenCV：维护 `scripts/process_work.py`，生成辅助图层。
- 标注与文案：维护 `annotation-draft.md` 和 `annotation.json`。
- 前端：维护 `web/`，实现图层、观察卡、探针、反思区。
- 答辩与文档：维护 README、任务书和演示流程。

不要多人同时修改同一个 JSON 文件。标注内容先写在草稿，再由一个人统一同步到 `annotation.json`。

## 7. 验收标准

演示时应能完成以下流程：

1. 打开 `http://localhost:5173/web/`。
2. 默认看到干净原作。
3. 切换 OpenCV 图层，说明它们是辅助证据。
4. 点击气脉、虚实、笔墨观察卡，左侧只高亮一处。
5. 右侧显示形式描述、感知经验、美学解释。
6. 点击图像任意位置，生成局部证据报告。
7. 用户在“我的观察”里写下自己的理解。
8. 明确说明系统不做自动审美判断。

## 8. 答辩表述

推荐表述：

> 我们的系统不是自动评价书法，而是把书法鉴赏中的抽象概念拆成可观察的证据。OpenCV 提供墨迹、骨架、粗细、留白等辅助图层；人工标注确认代表性观察点；用户可以通过图层切换、点击导览和局部探针逐步形成自己的理解。

不要说：

> 我们的系统可以自动识别气韵。

应该说：

> 我们的系统可以根据局部图像证据提示可能的观察候选，最后仍需要人工和用户解释。
