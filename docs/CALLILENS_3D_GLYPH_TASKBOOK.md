# CalliLens V1.7 三维字形提取与动态呈现任务书

## 1. 老师需求的重新理解

老师说“把字体从文本中提取，做成 3D 模式，加入 3D 动态效果”，在本项目里应解释为：

- 从书法作品图片中提取“单字墨迹字形”，不是生成 `.ttf` 电脑字体。
- 用 OpenCV 生成单字 mask、骨架和笔画厚度图。
- 用 Three.js 把单字做成可旋转的 3D 墨迹浮雕。
- 动态效果用于观察笔画骨架和墨迹厚薄，不代表真实笔顺识别。

## 2. 为什么不能直接全自动提取所有字

当前作品是行书长卷局部，存在以下问题：

- 字与字之间距离不稳定，部分笔画相互靠近或断裂。
- 行书有连带、飞白、枯笔，OpenCV 很难判断哪些墨迹属于同一个字。
- 印章、题名、正文大小差异明显，统一阈值容易误切。
- 仅靠传统 OpenCV 能提取墨迹轮廓，但不能稳定完成“汉字级语义分割”。

因此 V1.7 采用“人工框选字形 + OpenCV 提取 + Three.js 呈现”的稳妥方案。

## 3. 当前实现范围

- 默认作品：`work_003`。
- 默认精选 6 个字形：`宋`、`至`、`震`、`之`、`来`、`福`。
- 字框配置：`data/work_003/glyphs/glyph-boxes.json`。
- OpenCV 输出：`data/work_003/glyphs/glyphs.json`、`*_mask.png`、`*_height.png`、`*_skeleton.png`、`*_contours.json`。
- 前端入口：图层栏中的 `3D字形`。

## 4. 小组需要配合什么

标注同学只需要维护 `glyph-boxes.json`：

- `id`：英文或拼音唯一编号。
- `label`：展示用汉字。
- `annotationId`：可选，用来和人工导览卡联动。
- `description`：这个字为什么适合作为样例。
- `box`：原图百分比坐标。
- `tracePath`：可选，3-6 个相对坐标点，用于动态扫光。没有它也能展示，但不能说是真实笔顺。

每次改完字框后运行：

```powershell
python scripts/extract_glyphs.py --work data/work_003
```

## 5. 答辩口径

可以说：

> 我们用 OpenCV 从人工框选的书法字形中提取墨迹 mask、骨架和厚度图，再用 Three.js 生成三维墨迹浮雕。这样新手能从立体角度观察笔画粗细、墨迹重心和字内运动线索。

不要说：

> 系统自动识别了所有汉字。

> 系统恢复了真实笔顺。

> 系统生成了可安装字体。

> 系统能评价书法水平。

## 6. 验收标准

- `python scripts/extract_glyphs.py --work data/work_003` 能成功生成 6 个字形资产。
- `/web/` 能进入 `work_003`。
- 点击 `3D字形` 后原作仍可见，Three.js 展示单字墨迹浮雕。
- 点击字形按钮可以切换模型。
- 点击关联观察卡时，可以同步切换到相应字形。
- 页面文字说明必须强调“辅助观察，不代表真实笔顺或自动评分”。
