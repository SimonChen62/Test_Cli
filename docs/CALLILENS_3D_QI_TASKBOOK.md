# CalliLens Three.js 三维气韵辅助观察任务书

## 1. 本轮目标

本轮把 CalliLens 的“三维气韵”从装饰性 3D 效果改成可答辩的空间观察层。原作始终是主视图，Three.js 只把人工标注的气脉、虚实、笔墨线索转换成空间关系模型，帮助新手理解“为什么这里适合看气脉、哪里体现虚实、笔墨重心如何变化”。

核心口径：算法和 3D 只辅助观察，不替代人工解释，不输出书法评分。

## 2. 三维表达规则

- 气脉：用细曲线和流动点表示上下承接、形断势连和观看路线。动态描摹表示“势的延续”，不是自动识别出的真实运笔顺序。
- 虚实：用半透明空间片和呼吸式边界表示留白区域参与结构。它强调空间关系，不把留白解释成唯一审美答案。
- 笔墨：用小型重心柱和环形标记表示粗细、墨色、视觉重量的候选线索。它不是力度检测。
- 选中观察卡时，三维层只强化当前观察点，其他线索降低透明度，避免新手被过多信息干扰。

## 3. 前端与数据边界

- 默认作品仍为 `work_003`。
- 继续读取 `data/work_003/annotation.json`，不改变组员已经标注的坐标和解释内容。
- Three.js 场景只使用 `annotation.path` 和 `annotation.box` 生成空间对象。
- 原作图片、二维虚线高亮、局部探针、反思输入、专家反馈全部保留。
- Three.js 加载失败时只隐藏空间窗，其他功能必须继续可用。

## 4. 小组分工

- 前端：维护三段式流程、图层按钮、Three.js 空间窗和响应式布局。
- 标注：继续完善 `annotation.json` 中的人工观察点，尤其是气脉路径和虚实区域。
- 视觉验证：检查三维线索是否和二维标注位置一致，确认不会遮挡原作。
- 答辩：强调“人工导览 + 可见证据 + 三维辅助观察”，避免说成自动鉴定或自动评分。

## 5. 验收标准

- `/web/` 能进入首页、作品库和 `work_003` 核心导览页。
- 点击“三维气韵”后，原作图片仍然可见。
- 三维窗口不再显示马赛克式原图贴图，而是显示气脉曲线、虚实片和笔墨重心点。
- 点击不同人工导览卡时，二维标注和三维高亮同步变化。
- 右侧说明文字清晰可读，核心交互不回归。

## 6. 答辩表述

可以说：

> 我们用 Three.js 做的是三维辅助观察层。它把人工确认的气脉、虚实、笔墨观察点转成空间模型，让新手更容易理解书法中“势、留白、轻重”之间的关系。

不要说：

> 系统自动判断气韵好坏。

> Three.js 自动识别了真实运笔顺序。

> 这个模型可以评价书法水平。
