# CalliLens V1.6 任务书：入口页、作品选择与上传预留

## 1. 版本定位

V1.6 的目标不是把系统扩展成完整平台，而是把当前单作品 Demo 包装成一个更像真实产品的学习入口：

- 用户先看到 CalliLens 的入口页。
- 用户可以选择“浏览已存作品”，进入当前已经完成的书法导览流程。
- 用户可以看到“上传个性化书法作品”的入口，但 V1.6 暂不做真实上传后端。
- 系统继续坚持“算法辅助观察，人工确认解释，保留审美开放性”。

当前正式样例仍然是 `work_003`：赵孟頫《行书光福重建塔记卷》右段题名与正文局部。

## 2. 为什么暂不做 MySQL 和账号密码

课程 Demo 当前最重要的是说明交互逻辑和审美学习路径。MySQL、账号密码、权限、上传安全和部署会显著增加复杂度，但不会直接提高老师看到的核心价值。

V1.6 不做：

- 用户注册和登录。
- 密码存储。
- MySQL 数据库。
- 云端文件上传。
- 自动书法评分。
- 自动识字判断气脉。

V1.6 保留本地浏览器存储：

- 第一印象。
- 观察点反思。
- 当前作品选择。

## 3. 前端功能范围

### 3.1 首页入口

默认访问 `http://localhost:5173/web/` 时先进入入口页，而不是直接进入导览界面。

首页包含：

- 大标题：`CalliLens`
- 副标题：面向普通人的中国书法观察式导览
- 一段简短说明：先观察原作，再查看证据层、人工导览点和反思任务
- 向下滚动后出现两个主要按钮：
  - `上传个性化书法作品`
  - `浏览已存作品`

### 3.2 已存作品选择

点击 `浏览已存作品` 后显示作品卡片列表。

V1.6 的作品列表来自：

```text
data/works.json
```

每个作品卡片展示：

- 作品标题
- 类型或书体
- 简短说明
- 进入按钮

点击 `进入导览` 后进入现有导览页面，并加载对应作品数据。

### 3.3 上传入口占位

点击 `上传个性化书法作品` 后显示一个占位面板：

- 说明 V1.6 暂不上传到服务器。
- 说明后续流程会包括：上传图片、生成 OpenCV 图层、人工确认观察点、生成导览。
- 提供“返回入口”和“先体验已存作品”两个按钮。

这个入口用于答辩时说明系统未来扩展方向，但不假装已经完成后端。

## 4. 数据接口

新增作品索引：

```json
{
  "defaultWorkId": "work_003",
  "works": [
    {
      "id": "work_003",
      "title": "赵孟頫《行书光福重建塔记卷》右段题名与正文局部",
      "style": "行书（亦常称行楷）",
      "description": "包含正文、题名与印章的右侧局部，适合观察气脉、虚实和笔墨节奏。",
      "thumbnail": "original.png",
      "status": "ready"
    }
  ]
}
```

作品详细导览数据仍然使用：

```text
data/<workId>/annotation.json
```

## 5. 后端路线

### V1.6

不新增后端。

### V2 建议

如果后续必须做上传和保存，建议优先使用：

- FastAPI
- SQLite
- 本地 `uploads/` 文件夹

V2 数据表建议：

- `works`：作品元数据和图片路径
- `generated_layers`：OpenCV 图层路径
- `visitor_notes`：用户第一印象和反思文本
- `annotations`：人工确认后的观察点

只有当项目真的需要多人账号和长期部署时，再考虑 MySQL。

## 6. 验收标准

- 访问 `/web/` 默认看到入口页。
- 点击 `浏览已存作品` 能看到 `work_003`。
- 点击 `work_003` 能进入现有导览界面。
- 访问 `/web/?work=work_003&view=demo` 或从作品卡进入时，现有导览功能保持可用。
- 第一印象、观察卡、图层切换、探针、反思提交都不报错。
- 点击 `上传个性化书法作品` 显示明确的占位说明，不误导用户以为已经真实上传。

## 7. 小组分工建议

- 前端同学：完善入口页视觉、作品卡片和上传占位交互。
- 数据同学：维护 `data/works.json` 和每个作品的 `annotation.json`。
- 标注同学：继续修正 `work_003` 的观察点位置和解释文本。
- 后端同学：暂时不用搭 MySQL，先阅读本任务书的 V2 后端路线，准备 FastAPI + SQLite 原型即可。

## 8. 答辩表述

推荐说法：

> V1.6 把原来的单作品 Demo 扩展成一个作品入口系统。用户可以从已存作品进入导览，也可以看到未来上传个人作品的入口。当前版本为了控制范围，不做账号和数据库，而是集中验证书法鉴赏学习流程：原作第一印象、OpenCV 证据层、人工导览点、局部探针和反思表达。

不要说：

- 系统已经能自动分析任何书法作品。
- 系统可以自动判断气韵好坏。
- 系统能自动识别所有文字或真实笔顺。
- 上传功能已经完整完成。
