# Contributing

This repository is in an early planning stage. Keep changes small, traceable, and connected to an Issue whenever possible.

## Requirements

- Use Issues for requirements, ideas, bugs, and tasks.
- Each requirement should include background, expected behavior, and acceptance criteria.
- If the requirement is unclear, discuss it before starting implementation.

## Branches

- Use `main` as the stable branch.
- Create branches from `main`.
- Recommended branch names:
  - `feature/short-name`
  - `fix/short-name`
  - `docs/short-name`
  - `chore/short-name`

## Commits

Use short, clear commit messages.

Examples:

```text
Add initial login page
Fix form validation error
Update setup instructions
```

## Pull Requests

- Open a Pull Request for code changes.
- Link the related Issue when one exists.
- Explain what changed and how it was tested.
- Keep Pull Requests focused on one logical change.

## Safety

- Never commit secrets, tokens, passwords, or private keys.
- Avoid committing large binary files, datasets, generated outputs, or dependency folders.
- If sensitive data was committed by mistake, notify the repository owner immediately.
