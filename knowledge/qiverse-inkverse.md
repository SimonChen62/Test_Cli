# QiVerse / InkVerse 轻量实验说明

## 核心定位

QiVerse / InkVerse 不是把书法作品挂进虚拟展厅，而是让观众进入笔画、沿笔势穿行、追随形断势连的关系，并走入留白空间，最后回到原作重新观看。

当前 CalliLens 项目中的实验版本命名为 QiVerse Lite。它是一个桌面优先、WebXR 可增强的 Three.js 原型，用于验证老师文档中的核心体验链。

## 老师文档的核心体验链

Original：先看原作。

Enter the Ink：进入墨迹，重墨变成更厚的墨壁，飞白和细线变成更薄、更碎的颗粒。

Ride the Stroke：相机沿关键笔画路径移动，让观众感到“一笔不是静止的线，而是发生过的运动”。

Follow the Qi：用淡线和粒子表现形断势连，避免把气脉画成粗箭头。

Walk into the Void：把留白转化为可感知的空间，包括字内空腔、字间通道、行间开放区域。

Return：回到原作，让用户重新观察运动、断续、留白和整体呼吸。

## 当前实验分支实现范围

当前分支先实现 QiVerse Lite：

1. 独立页面 `qiverse/index.html`。
2. Three.js 场景和体验状态机。
3. 人工解释性数据模型 `qiverse/calligraphy-work.json`。
4. 原作平面、3D ink ribbon、气脉粒子、留白空间块。
5. Desktop Mode 完整可用。
6. 浏览器支持时显示 WebXR 入口。

## 为什么先做轻量版

完整 WebXR、真实 VR 设备、声音、复杂 shader 和自动笔画识别会让项目风险迅速变大。当前课程展示更需要一个能运行、能说明、能答辩的闭环版本。

因此第一版不做：

1. 不自动恢复真实笔顺。
2. 不自动判断真正气韵。
3. 不做书法水平评分。
4. 不要求 VR 设备才能体验。
5. 不把随机粒子当作书法分析。

## 和 RAG 的结合

RAG 负责解释体验节点和资料来源，例如：

用户问“为什么要进入留白？”

系统应基于本地知识库回答：留白不是空无，而是参与字内、字间、行间结构和节奏组织。QiVerse Lite 用空间块把这件事可视化，但它是解释性展示，不是自动审美判断。

用户问“Follow the Qi 是不是算法识别气韵？”

系统应回答：不是。当前版本是 expert-informed interpretive visualization，即基于人工标注和图像特征的解释性可视化。
