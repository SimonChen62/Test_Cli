# QiVerse / InkVerse 电影体验说明

## 核心定位

QiVerse / InkVerse 不是把书法作品挂进虚拟展厅，而是让观众经历一次“进入书法内部”的观看过程：先看见散落星尘，再看见星点聚成赵孟頫《光福重建塔记》，随后进入墨迹、沿笔势穿行、追随形断势连的气脉提示、走入留白，最后回到原作。

当前 CalliLens 项目中的实验版本命名为“QiVerse 墨气星河”。它是桌面优先、WebXR 可增强的 Three.js 前端体验，用于验证老师文档中的核心体验链是否能被普通观众理解。

## 老师文档的核心体验链

Silence / 静墨：体验开始前，画面中只有散落星点，观众还没有进入文字。

Enter the Ink / 入墨：星点聚成原作墨迹，镜头贴近墨迹边缘，墨色较重的位置在 Z 轴上更突出。

Ride the Stroke / 御笔而行：镜头沿墨迹点云移动，表现线条曾经是动作，而不是静止图案。

Follow the Qi / 追势：用淡金色粒子跨越墨迹断口，提示“形断势连”。这是一种解释性可视化，不是算法自动判断气韵。

Walk into the Void / 入白：墨迹淡出，字内、字间、行间留白区域显现，让观众意识到没有被写出的地方也在组织节奏。

Return / 回看原作：体验结束后回到原作长卷，让观众带着运动、断续、留白和整体呼吸的感受重新观看。

## 当前分支实现范围

当前分支 `codex/qiverse-cinematic-flow` 实现的是前端电影式体验：

1. 独立页面 `qiverse/index.html`。
2. Three.js 全屏场景。
3. 从 `data/work_003/original.png` 采样深色像素，生成墨迹点云。
4. 星河散点到书法长卷的插值动画。
5. 入墨、御笔、追势、入白、回看的章节按钮。
6. 少量人工解释性 `qiLinks` 和 `voidRegions` 数据。

它不是正式 WebXR 大工程，也不要求 VR 设备才能演示。

## 和 RAG 的结合

RAG 负责解释体验节点和项目边界。例如：

用户问“QiVerse 和 CalliLens 怎么结合？”

系统应回答：CalliLens 是作品展示、RAG 导览、用户记录和管理员管理层；QiVerse 是沉浸式体验层。QiVerse 可以从 CalliLens 的作品图像和标注数据中读取材料，再用 Three.js 做星河点阵、入墨、追势和入白效果。

用户问“Follow the Qi 是不是算法识别气韵？”

系统应回答：不是。当前版本是基于人工解释性标注和图像特征的展示，不声称自动判断气韵、书法水平、真伪或真实笔顺。

用户问“为什么要用星点聚成书法？”

系统应回答：星点聚合可以把“从远观到入墨”的过程做成清晰的视觉叙事。观众先看到散落的视觉材料，再看到它们归位为原作墨迹，最后进入墨迹内部观察运动和留白。

## 为什么先做电影式前端

完整 WebXR、真实 VR 设备、声音、复杂 shader 和自动笔画识别都会让项目风险迅速变大。当前课程展示更需要一个能运行、能看懂、能答辩的版本，所以先做桌面浏览器可演示的电影式前端体验。

后续如果老师认可，再升级为：

1. Vite + TypeScript 独立工程。
2. 管理员可视化标注 qiLinks 和 voidRegions。
3. 60-90 秒自动演示视频模式。
4. WebXR / Meta Quest Browser 真机测试。
5. 行为日志和反思记录接入。

## 边界说明

当前版本不做：

1. 不恢复真实笔顺。
2. 不自动判断气韵。
3. 不评价书法好坏。
4. 不判断作品真伪。
5. 不把随机粒子当成学术分析。

推荐答辩表述：

> QiVerse 墨气星河把原作中可见的墨迹采样成点云，通过星点聚合、入墨穿行、追势和入白的前端动画，让观众从空间和运动角度重新理解赵孟頫《光福重建塔记》。RAG 负责解释这些体验节点的文化含义和技术边界，避免把视觉效果夸大成自动书法鉴定。
