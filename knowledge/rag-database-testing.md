# RAG 知识库、数据库与测试说明
source: 项目技术说明

## 现在有数据库吗
当前版本没有 MySQL、SQLite、向量数据库，也没有云端数据库。CalliLens 第一版使用本地文件型知识库，优点是新手容易运行、无需安装数据库、课堂演示更稳定。

## RAG 数据库在哪里
当前 RAG 的“数据库”实际是本地文件集合：作品列表在 `data/works.json`，默认作品知识片段在 `data/work_003/knowledge.json`，通用知识库在 `knowledge/*.md`。后端每次回答问题时读取这些文件，做关键词和标签匹配，再返回答案和来源。

## 为什么现在不用向量数据库
当前资料量很小，关键词检索已经能覆盖赵孟頫、《光福重建塔记》、书法术语、OpenCV、Three.js、RAG 和 InkVerse 任务说明。向量数据库适合资料量更大、作品更多、语义检索要求更高的后续版本。

## 做完之后怎么测试
先运行 `python -m pip install -r requirements.txt` 安装依赖，再运行 `.\start-demo.ps1`。浏览器打开 `http://127.0.0.1:5190/web/` 看前端，打开 `http://127.0.0.1:8000/docs` 看后端接口。RAG 可以问“现在有数据库吗？”、“RAG 数据库在哪里？”、“老师新文档要求是什么？”。

## 没有 API key 能不能运行
可以。默认模式是本地 RAG 检索加模板化回答，不需要付费 API。管理员配置 API key 后，大模型只负责润色检索到的资料，不负责凭空补充事实。

## 管理员上传后会发生什么
管理员上传作品后，系统会保存原图，运行图像处理，生成 mask、height、缩略图和作品知识片段，并更新 `data/works.json`。新作品会进入作品库，也能被本地 RAG 检索。
